# Процедурные туманности — дизайн-спецификация

**Дата:** 2026-06-26
**Статус:** утверждён к реализации
**Модуль:** `src/core/renderables/Nebula/`

## 1. Цель и контекст

Система генерации процедурных объёмных туманностей для космической симуляции Titan Engine
(TypeScript + Three.js `^0.182` + React + MobX). Туманность — это «осязаемый» объект сцены, а
не часть скайбокса: камера свободная, сцена бесшовная, камера может оказаться как снаружи, так и
внутри туманности, и в обоих случаях эффект должен выглядеть корректно и красиво.

Туманности покрывают преимущественно локальные области сцены (астероидные кластеры, планетные
системы), но размер сильно параметризуем — от малых сгустков до крупных облаков. Баланс
реализм/зрелищность — 50/50; научная достоверность не требуется, но формы должны быть
правдоподобно асимметричными. Референсы: Star Citizen, Elite Dangerous.

## 2. Зафиксированные решения

- **Адаптивный LOD.** Один объект `Nebula` покрывает весь спектр железа и дистанций: вблизи / на
  мощном GPU — честный реймарчинг, вдали / на слабом — билборд-импостер с кэш-текстурой.
- **Размер** сильно параметризуем; оба режима камеры (внутри / снаружи) равноважны.
- **Освещение** самосветящееся (мультихромная эмиссия) + дешёвый направленный рассеиватель от
  ближайшей звезды. **Без light-marching.** Весь свет в проекте белый — цвет источника из
  уравнения исключён.
- **Форма статична** (динамические течения газа — для демо-сцен, не для реализма). Это позволяет
  надолго кэшировать импостер: перепекание только при заметном изменении ракурса (параллаксе).
- **Автономный типизированный API.** На этапе разработки: `new Nebula(params)` → `Object3D`,
  вставляется через `scene.add(...)`, без связи с `Actor`/сценариями. Интеграция с моделью данных
  движка — отдельным адаптером позже, когда вид будет признан достаточно проработанным.

### Целевая платформа

`logarithmicDepthBuffer: true`, near `1e-6`, far `2000 а.е.`; постпроцессинг через библиотеку
`postprocessing` (`EffectComposer`, depth-буфер включён, `HalfFloatType`, multisampling 8);
`BloomEffect` с `luminanceThreshold: 1` (HDR-эмиссия > 1 уходит в bloom автоматически);
`SpaceScale ≈ 5·10⁻⁴` (≈ 2000 км на 1 Three.js-юнит).

## 3. Подход к рендерингу

Три способа «потребить» единый конвейер плотности под бюджет кадра:

- **A — аналитический реймарчинг процедурной плотности (ядро).** Плотность = warped FBM в
  локальном пространстве bounding-прокси; вблизи марчим, вдали — импостер. Композит на уровне поля
  (несколько лобов плотности в одном проходе).
- **C — запечённый 3D-текстурный объём (опциональная оптимизация).** Статичную форму один раз
  печём в 3D-текстуру и марчим дешёвым сэмплингом вместо тяжёлого FBM. Off по умолчанию; включается
  для тяжёлых туманностей. Риск — память.
- **B — слоистый билборд/импостер (дальний LOD / фолбэк).** Самый дешёвый уровень для малого
  экранного размера и слабого железа.

Рекомендованная и принятая комбинация: ядро **A** + **C** как профилируемая оптимизация + **B** как
дальний LOD.

## 4. Напряжения в требованиях (вскрыты намеренно)

1. **«Дёшево» ↔ «камера внутри и красиво».** Плоские билборды разваливаются изнутри. Решение —
   адаптивный LOD: честный объём вблизи, импостер вдали. Цена переносится в бесшовность перехода
   (главный риск).
2. **«Композит из многих» ↔ «дёшево».** `Group` из N реймарчащихся мешей даёт взрывной overdraw
   изнутри. Решение — композиция на уровне поля плотности (`lobes[]` в одном проходе), а не на
   уровне рендера. `Group` остаётся возможным, но не является механизмом композиции по умолчанию.
3. **«Реализм асимметрии» ↔ простота шума.** Чистый FBM даёт «пушистый шарик». Нужны domain warp +
   ridged/billow + Worley-каверны + лобы.

## 5. Архитектура и раскладка модулей

Самодостаточный модуль — всё, включая материалы и шейдеры, под `renderables/Nebula/`:

```
src/core/renderables/Nebula/
  index.ts                     // export { Nebula }
  Nebula.ts                    // Object3D-контейнер; updateObject(delta); LOD-свитч
  NebulaParams.ts              // типы + DEFAULT_NEBULA_PARAMS
  volume/
    NebulaVolume.ts            // Mesh: bounding-прокси + реймарч-материал (LOD A, опц. C)
    NebulaImpostor.ts          // Mesh: камера-facing билборд с кэш-текстурой (дальний LOD)
    ImpostorBaker.ts           // RTT по порогу параллакса
  material/
    NebulaRaymarchMaterial.ts  // extends AbstractShaderMaterial
    NebulaRaymarchShader.ts    // extends AbstractShader → ShaderProps
    NebulaImpostorMaterial.ts  // дешёвый материал билборда
    shader/
      raymarch.template.ts     // ShaderProps: uniforms + vertex + fragment
      chunks/
        NebulaNoise.ts         // FBM / ridged / billow / domain warp (поверх <noiseFunctions>)
        NebulaDensity.ts       // density(p): лобы + каверны + dust
        NebulaColor.ts         // мультихромная палитра + дешёвый свет
        NebulaDepth.ts         // линеаризация log-depth + soft-intersect
  fields/
    NebulaField.ts             // CPU-зеркало плотности (размещение лобов + юнит-тесты)
    SeededRandom.ts            // паттерн из DetailedRingStreamingSystem
  presets/
    index.ts                   // emission / reflection / dark пресеты
```

### Конвенции, на которые опираемся

- **Шейдеры** через паттерн `ShaderProps` + `AbstractShader` (`#include <chunk>` →
  `AppShaderChunk`). Переиспользуем существующий `#include <noiseFunctions>` (`snoise(vec3)`) как
  общую либу; FBM/ridged/warp — композиции, живут в чанках туманности.
- **Материал** наследует `AbstractShaderMaterial` (`updateMaterial`/`resetMaterial`).
- **Тик** — `updateObject(delta)`: `SceneManager.update()` делает
  `scene.traverse(o => o.updateObject(delta))`, т.е. добавленный в сцену объект тикается
  автоматически. На этапе разработки допустим ручной вызов из тестового `update`.
- **Масштаб** — `size` в Three.js-юнитах через `toThreeJSUnits`, внутренняя математика — в
  нормированном локальном пространстве прокси.
- **Worley** — переиспользуемый примитив `cellular3d(vec3) → vec2(F1, F2)` добавляется в общий
  `Noise.ts` (явный шаг плана, требует одобрения владельца).

## 6. Параметризация (`NebulaParams`)

```ts
interface NebulaParams {
  seed: number                    // детерминизм всей формы

  // — Форма / габариты —
  size: number                    // полу-размер в Three.js-юнитах (через toThreeJSUnits)
  shape: 'ellipsoid' | 'disk'     // тип bounding-прокси и boundary-спада
  axisRatios: Vec3                // анизотропия (вытянутость) — против «шарика»
  edgeFalloff: number             // мягкость внешней границы

  // — Внутренняя структура (асимметрия) —
  lobes: NebulaLobe[]             // композит на уровне поля: центр + радиус + вес + own-seed
  cavities: NebulaCavity[]        // выгрызенные пустоты (звёздный ветер)
  noise: {
    octaves: number               // детализация FBM (входит в LOD-бюджет)
    frequency: number
    lacunarity: number
    gain: number
    warpStrength: number          // domain warp → филаменты, асимметрия
    ridged: number                // 0..1 микс billow↔ridged (волокнистость)
    contrast: number              // gamma плотности: дымка ↔ плотные сгустки
  }

  // — Цвет (мультихромность) —
  palette: {
    stops: ColorStop[]            // градиент по плотности (мин. 2 цвета)
    secondary: Color              // «ионизационный» 2-й канал эмиссии
    secondaryThreshold: number    // где проступает 2-й цвет
    emissiveIntensity: number     // HDR; >1 уходит в bloom движка
  }

  // — Пыль (тёмные прожилки поглощения) —
  dust: {
    strength: number              // сила поглощения
    threshold: number             // где включается пылевая полоса (доп. ridged/Worley-канал)
    color: Color                  // тёмный тон прожилок (по умолчанию ~чёрно-бурый)
  }

  // — Свет (дешёвый рассеиватель, белый) —
  lighting: {
    starPosition: Vec3 | null     // позиция ближайшей звезды (мир)
    scatterStrength: number       // направленное подсвечивание
    ambient: number               // базовая видимость в тени
  }

  // — Рендер / производительность —
  quality: {
    maxSteps: number              // потолок сэмплов реймарча
    resolutionScale: number       // 0.25..1 (полный res сейчас, half-res позже)
    forceLOD?: 'raymarch' | 'impostor' | 'auto'
    bake3DTexture: boolean        // оптимизация C (off по умолчанию)
    bakeResolution: number        // 64..256
  }
}
```

- `lobes[]` — главный инструмент композита: «хорошая одна туманность без Group» достигается
  богатством лобов/каверн.
- `DEFAULT_NEBULA_PARAMS` рисует осмысленную туманность без аргументов; пресеты переопределяют
  группы полей.
- `quality.*` — ручки LOD, связанные с адаптивным контроллером; на слабом железе / вдали зажимаются
  автоматически.

## 7. Модель плотности и внешнего вида

`density(p)` (один свод правил для GPU-реймарча и CPU-`NebulaField`), слоями от крупного к мелкому:

1. **Boundary** — аналитический спад по `shape` + `axisRatios` (эллипсоид: `1 - smoothstep` по
   нормированному радиусу; диск: радиальный спад × вертикальный сквош). Замкнутый силуэт и корректная
   работа изнутри.
2. **Лобы** — сумма гауссиан/метаболов из `lobes[]` модулирует базу (несимметричный многоцентровый
   силуэт).
3. **Domain warp** — `p += warpStrength * fbm(p)` перед основным шумом (потёки, филаменты).
4. **FBM** — `octaves`/`lacunarity`/`gain`, микс billow↔ridged по `noise.ridged`.
5. **Каверны (Worley)** — один низкочастотный слой `cellular3d`; `F2-F1` режет стенки-филаменты,
   `F1` выгрызает пустоты. Подмешивается мультипликативно. **Вне октавного цикла FBM** (контроль
   цены).
6. **Контраст** — `pow(density, contrast)`. Итог клампится в `[0,1]`, домножается на `edgeFalloff`;
   вне прокси строго `0` (ранний выход марча).

**Цвет / мультихромность:** `palette.stops` — LUT по плотности (края ≠ ядро); `palette.secondary` +
`secondaryThreshold` — второй эмиссионный канал ионизации (Hα-ядро vs OIII-края). Эмиссия в HDR,
`emissiveIntensity > 1` сознательно цветёт через существующий BloomEffect.

**Пыль:** отдельный канал поглощения — там, где маска (низкочастотный ridged/Worley) выше
`dust.threshold`, гасит накопленную эмиссию и повышает extinction вдоль луча. Тёмные лейны
(Конская Голова).

**Свет (белый, без light-marching):** на сэмпле направленный член
`scatter = scatterStrength * pow(max(dot(viewDir, toStar), 0), k)` + ослабление по расстоянию до
`starPosition`; `ambient` — видимость в тени. Самозатенения нет (форма статична).

**Накопление** — emission-absorption front-to-back, premultiplied alpha (как `StarOuterLayer`),
ранний выход при насыщении alpha.

## 8. Рендер-конвейер, LOD и глубина

- **Прокси-геометрия.** Bounding-меш (эллипсоид → box/икосфера по `axisRatios`; диск → сплющенный
  box), рендер задних граней (`side: BackSide`, `depthWrite: false`). Начало луча — камера, конец —
  выход из прокси. Корректно снаружи и изнутри, бесплатный frustum culling, нет марча вне покрытия.
- **Марш.** В нормализованном локальном пространстве (шаг не зависит от мирового масштаба);
  `quality.maxSteps`, dithering старта луча (bayer/синий шум) против бандинга, ранний выход по alpha
  и по выходу из прокси.
- **Глубина (`logarithmicDepthBuffer`).** Чанк `NebulaDepth`: реконструкция линейной глубины из
  лог-буфера (не стандартный `perspectiveDepthToViewZ`); soft-intersect (плавное гашение плотности
  у твёрдой геометрии); ограничение дальнего конца луча глубиной сцены.
- **LOD-контроллер (`updateObject`).** По проективному экранному радиусу прокси: крупный → реймарч
  на верхней планке; средний → реймарч с зажатыми сэмплами (опц. источник — 3D-текстура, режим C);
  малый/далёкий/слабое железо → `NebulaImpostor`. `forceLOD` перекрывает авто. Переход
  реймарч↔импостер — **кроссфейд по экранному размеру с гистерезисом** (без мерцания на границе).
- **Импостер.** `ImpostorBaker` рендерит туманность во `WebGLRenderTarget` тем же реймарч-материалом
  один раз; билборд показывает текстуру. Перепекание только при изменении ракурса больше порога
  параллакса (форма статична → вдали почти нет перепеканий).
- **Блендинг.** `transparent: true`, `depthWrite: false`, premultiplied-alpha emission-absorption.
  Рендер в основном проходе сцены до постпроцессинга → bloom/tonemapping применяются бесплатно.

### Согласованные компромиссы

- **Разрешение реймарча:** полный res + dithering сейчас; half-res с depth-aware апсемплом —
  оптимизация второй итерации.
- **Прозрачность vs прозрачность:** реймарч корректно перекрывается с непрозрачной геометрией через
  depth; порядок между несколькими полупрозрачными туманностями / коронами — известное ограничение
  (ещё один довод за композит лобами, а не Group).

## 9. Тестирование

WebGL в jsdom недоступен — стратегия двухуровневая:

- **Юнит-тесты `NebulaField` (CPU-зеркало плотности):** детерминизм по `seed`; `density ∈ [0,1]`;
  вне прокси строго `0`, в центре без каверн > 0; `disk` сплюснут по Y сильнее `ellipsoid` при равных
  `axisRatios`; добавление лоба повышает плотность в окрестности; композит из N лобов несимметричен;
  каверны/dust повышают изрезанность (дисперсия/градиент поля).
- **Чистые функции параметров:** `DEFAULT_NEBULA_PARAMS` валиден; пресеты мёржатся; клампы
  `quality.*` работают.
- **Smoke-тест конструктора:** `new Nebula(params)` строит корректную иерархию `Object3D` с
  мок-three, без реального GL.
- **GPU/визуал — вручную** циклом обратной связи (`scene.add` → смотреть → фидбэк). Отладочная панель
  параметров — позже.

CPU-зеркало нужно по делу (размещение лобов/каверн при конструировании); тестируемость — бонус.
GLSL- и TS-версии плотности держатся синхронно по одному своду правил.

## 10. Реестр рисков

| Риск | Митигация |
|---|---|
| Бесшовность перехода реймарч↔импостер (главный) | кроссфейд по экранному размеру + гистерезис; импостер тем же материалом |
| Реконструкция глубины из `logarithmicDepthBuffer` | изолирована в `NebulaDepth`; отлаживается на простой сцене |
| Overdraw при влёте камеры внутрь | марч в локальном пространстве, ранний выход по alpha/прокси; композит лобами |
| FBM×Worley дорог на сэмпл | Worley вне октавного цикла; `octaves`/`maxSteps` в LOD-бюджете; путь C для тяжёлых |
| Память 3D-текстур при многих композитах | bake опционален и off по умолчанию; разрешение 64–256 |
| Сортировка прозрачных туманностей между собой | известное ограничение; композит лобами как основной путь |
| GLSL↔CPU плотность расходятся | общий свод правил; юнит-тесты на CPU фиксируют инварианты формы |

## 11. Порядок реализации

Каждый шаг даёт видимый на сцене результат:

1. Прокси-меш + базовый boundary-реймарч (один цвет) — объём виден снаружи и изнутри.
2. FBM + domain warp + контраст — форма и асимметрия.
3. Мультихромная палитра + 2-й канал + dust.
4. Дешёвый направленный свет + интеграция глубины (soft-intersect).
5. Лобы + каверны (Worley, добавить `cellular3d` в `Noise.ts`) — композит в одном объекте.
6. LOD: импостер + кроссфейд.
7. (Опц.) 3D-текстурный bake; (опц.) half-res.

## 12. Вне рамок этой спецификации

- Интеграция с `Actor` / сценариями / visitor-пайплайном (отдельный адаптер позже).
- MobX-стор и UI-панель «живых ручек» (после стабилизации вида).
- Динамические течения газа / анимация формы (сознательно отвергнуты ради реализма).
- Цветное освещение от звезды (весь свет белый).
- Light-marching / честное самозатенение объёма.
