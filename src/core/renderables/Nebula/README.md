# Nebula — процедурная объёмная туманность

Самостоятельный `Object3D`: `new Nebula(params)` → `scene.add(...)`. Объёмный
raymarch вблизи, дешёвый запечённый импостер-билборд вдали (адаптивный LOD).
Форма **статична** — это позволяет запекать поле плотности в 3D-текстуру.

```ts
import { Nebula } from '@/core/renderables/Nebula'
import { Vector3 } from 'three'

const nebula = new Nebula({ size: 2000 })
scene.add(nebula)
// при удалении:
nebula.dispose()
```

> **Важно про lifecycle:** параметры применяются **в конструкторе**. Рантайм-сеттера
> пока нет — чтобы изменить параметры, пересоздай `new Nebula({...})`. Привязывай
> туманность к стабильному узлу (группе), а **не** к мешу, который может подменяться
> родительским `THREE.LOD`, иначе она исчезнет вместе с мешем.

---

## Пресеты

```ts
import { makeNebulaParams } from '@/core/renderables/Nebula/presets'

new Nebula(makeNebulaParams('emission'))            // самосветящаяся, ярче
new Nebula(makeNebulaParams('reflection'))          // тусклее, лит звездой (низкий ambient)
new Nebula(makeNebulaParams('dark'))                // плотная пыль, контраст
new Nebula(makeNebulaParams('emission', { size: 5000, seed: 7 })) // пресет + оверрайды
```

---

## Параметры

`mergeNebulaParams(overrides)` глубоко сливает оверрайды с дефолтами; всё
опционально. Координаты лобов/каверн — в **локальном пространстве `[-1,1]`**.

### Форма

| Параметр | Дефолт | Эффект |
|---|---|---|
| `seed` | `1337` | зерно шума; меняет конкретный рисунок при той же форме |
| `size` | `1000` | полу-размер прокси в мировых единицах (габарит туманности) |
| `shape` | `'ellipsoid'` | `'ellipsoid'` или `'disk'` (диск сплюснут по Y) |
| `axisRatios` | `(1, 0.8, 1)` | анизотропия эллипсоида (x,y,z); применяется один раз внутри шейдера |
| `edgeFalloff` | `0.35` | мягкость кромки; ↑ → более растушёванный полупрозрачный край (0..~0.6) |
| `density` | `4.0` | **оптическая плотность/непрозрачность** (поглощение на шаг); ↓ → прозрачнее. Главная ручка «ватность ↔ воздушность» |

### Шум / структура (`noise`)

| Параметр | Дефолт | Эффект |
|---|---|---|
| `octaves` | `5` | детализация fbm; ↑ → мельче детали (дороже на живом пути) |
| `frequency` | `1.6` | базовая частота; ↑ → мельче клочья |
| `lacunarity` | `2.0` | множитель частоты между октавами |
| `gain` | `0.5` | спад амплитуды между октавами |
| `warpStrength` | `0.35` | домен-варп (искажение); ↑ → более «текучая»/органичная форма |
| `ridged` | `0.4` | микс billow↔ridged (0=пухлые комки, 1=острые гребни/филаменты) |
| `contrast` | `1.6` | резкость спада плотности; ↑ → больше пустот, «воздушнее» |
| `worleyStrength` | `0.4` | Worley-каверны (ячеистые филаменты); ↑ → сильнее прожилки. `0` отключает (экономит шаг на живом пути) |

### Лобы и каверны (композиция под-облаков в одном проходе)

```ts
new Nebula({
  lobes: [
    { center: new Vector3(0.4, 0.1, 0.2), radius: 0.5, weight: 1.2, seed: 1 },
    { center: new Vector3(-0.5, -0.2, 0.3), radius: 0.4, weight: 1.0, seed: 2 }
  ],
  cavities: [
    { center: new Vector3(0.1, 0, 0), radius: 0.4, strength: 0.7 } // выгрызенная полость
  ]
})
```
- **Лоб**: `center` (лок. [-1,1]), `radius`, `weight` (вклад плотности), `seed`. Даёт многоцентровую асимметрию (требование «составные группы» в одном raymarch-проходе).
- **Каверна**: `center`, `radius`, `strength` (0..1 — сила выгрызания).
- Максимум по 8 каждого; лишнее отбрасывается. Запекаются в 3D-текстуру → на марше бесплатны.

### Цвет (`palette`) — мультихромность

| Параметр | Дефолт | Эффект |
|---|---|---|
| `stops` | сине-зелёные (4 стопа) | градиент по плотности: `{ t: 0..1, color }`, от разрежённого к плотному |
| `secondary` | `#5aa0d8` | акцентный цвет, подмешиваемый в **плотных** областях (вторая хрома) |
| `secondaryThreshold` | `0.6` | с какой плотности начинается подмешивание secondary |
| `emissiveIntensity` | `1.6` | яркость самосвечения; `>1` → блумит (HDR) |

Дефолтные стопы: `#06141c` → `#1f6b66` → `#4cbfa6` → `#bdeede` (тёмно-бирюзовый →
бирюза → аква-зелёный → бледная мята).

### Пыль (`dust`) — затемнение

| Параметр | Дефолт | Эффект |
|---|---|---|
| `strength` | `0.6` | сила затемнения пылевыми полосами |
| `threshold` | `0.55` | порог пыли, выше которого темнит |
| `color` | `#05090c` | цвет затемнения (прохладный тёмный) |

### Свет (`lighting`) — дешёвый белый направленный

| Параметр | Дефолт | Эффект |
|---|---|---|
| `starPosition` | `null` | мировая позиция звезды; `null` → только самосвечение. Задай → односторонняя подсветка |
| `scatterStrength` | `0.8` | сила forward-scatter к звезде (аддитивно поверх самосвечения) |
| `ambient` | `1.0` | **база самосвечения** (не «заполняющий свет»). 1.0 = полностью видима без звезды. Reflection-пресет ставит низко |

Свет всегда **белый** (без цвета звезды) — по решению владельца.

### Качество / производительность (`quality`)

| Параметр | Дефолт | Эффект |
|---|---|---|
| `bake3DTexture` | `true` | запекать статичное поле в 3D-текстуру → 1 выборка/шаг вместо ~12 шумов (стабильные 144 FPS изнутри). ~2 МБ VRAM/туманность |
| `bakeResolution` | `128` | разрешение запекания (64–256). ↑ → чётче детали, больше памяти/времени старта |
| `maxSteps` | `64` | шагов raymarch; ↑ → меньше бандинга, дороже (важно только при `bake3DTexture:false`) |
| `forceLOD` | `'auto'` | `'auto'` / `'raymarch'` / `'impostor'` — форсировать уровень LOD (для отладки) |
| `depthTest` | `true` | `true` — непрозрачная геометрия перекрывает туман (но даёт жёсткие вырезы, напр. о bounding-сферу чёрной дыры); `false` — туман поверх всего. Бинарный выбор «меньшего зла» (см. ограничения) |
| `resolutionScale` | `1` | зарезервировано (half-res пока не реализован) |

## Рецепты тюнинга

- **Прозрачнее / менее «ватно»**: `density` ↓ (или, как ни странно, сильно ↑ — выше насыщение у поверхности), `contrast` ↑, `edgeFalloff` ↑.
- **Больше филаментов/прожилок**: `worleyStrength` ↑, `ridged` ↑.
- **Многоцентровое облако**: задать `lobes[]` (+ `cavities[]` для полостей).
- **Другая палитра**: заменить `palette.stops` и `secondary`. Яркие стопы × `emissiveIntensity>1` → блум.
- **Подсветка от звезды**: `lighting.starPosition` = мировая позиция звезды.
- **Сплюснутый диск**: `shape: 'disk'` (+ `axisRatios` для пропорций).
- Крути **по одной ручке за раз** — начни с `density` и `contrast`.

> После изменения формы (`bake3DTexture:true`) поле перепекается при пересоздании
> объекта. Чисто цветовые/световые ручки тоже применяются при пересоздании.

---

## Производительность

- **Форма статична** → `bake3DTexture` (вкл. по умолчанию) запекает плотность
  (boundary + warp + fbm + лобы + Worley) один раз в `Data3DTexture`; марш делает
  1 trilinear-выборку на шаг. Это снимает основную стоимость (≈144 FPS изнутри).
- **Импостер-LOD**: вдали туманность рендерится дешёвым запечённым билбордом,
  кроссфейд в полосе ~150–450 px проекционного радиуса.
- Worley тяжёлый на **живом** пути (`bake3DTexture:false`) — там можно отключить
  через `worleyStrength: 0`. При запекании это неактуально (всё в текстуре).

---

## Известные ограничения

- **Глубинное взаимодействие (11B)** — мягкое пересечение с объектами внутри
  туманности (астероиды/планеты) не реализовано: `depthTest` — бинарный компромисс.
  Чёрная дыра — особый случай (пишет глубину оболочки-сферы), корректно с туманностью
  не сосуществует без правок самой дыры; рекомендуется не смешивать в одной сцене.
- **Рантайм-тюнинг** — параметры применяются в конструкторе; для изменения
  пересоздавай объект (рантайм-сеттер — кандидат на будущее, в т.ч. под UI-слайдеры).
- `resolutionScale` / half-res — зарезервировано, не задействовано (bake решил перф).
